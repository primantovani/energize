Apresentação Arquitetura da Solução Diagrama Wireframe

Principais Componentes da Arquitetura

Interface do Usuário (Front-End)

Linguagens de Programação

O front-end é desenvolvido com HTML, CSS e JavaScript, que são fundamentais para criar páginas estruturadas, estilizadas e interativas. Essas tecnologias permitem que a interface do usuário seja visualmente atraente e funcional, garantindo uma experiência intuitiva e responsiva.

Framework Front-End

React.js foi escolhido devido à sua capacidade de criar componentes reutilizáveis e interativos. Isso facilita a construção de dashboards com gráficos e tabelas para monitoramento da energia gerada e consumida.

Ferramenta de Desenvolvimento

O Visual Studio Code é utilizado como IDE, oferecendo suporte a extensões úteis para depuração, integração com controle de versão e análise de código. Ele é altamente personalizável, adaptando-se às necessidades específicas do projeto.

Lógica do Sistema (Back-End)

Linguagem de Programação

Node.js é usado no servidor por sua alta escalabilidade e eficiência em lidar com múltiplas solicitações simultâneas, como os dados provenientes de dispositivos IoT.

Framework Back-End

O Express.js permite criar APIs para gerenciar e processar informações sobre geração de energia solar, consumo e economia de custos.

Banco de Dados

MongoDB foi escolhido como o banco de dados por sua flexibilidade em armazenar grandes volumes de dados estruturados e não estruturados. Ele é usado para:

  • Registrar dados de energia gerada e consumida.
  • Armazenar configurações dos dispositivos IoT conectados.
  • Salvar relatórios personalizados de economia e pegada de carbono.

Segurança e Autenticação

Métodos de Segurança

A aplicação implementa login e cadastro com criptografia de senhas, garantindo a proteção dos dados do usuário. Além disso, a verificação por e-mail adiciona uma camada extra de segurança, validando a identidade de cada usuário antes do acesso ao sistema.

Armazenamento em Nuvem com AWS S3

O sistema utiliza o AWS S3 (Amazon Simple Storage Service) como solução de armazenamento em nuvem, oferecendo escalabilidade, segurança e alta disponibilidade para os dados gerados e utilizados pela aplicação.

Principais Implementações do AWS S3

  • Armazenamento de Relatórios Personalizados: Relatórios mensais em formato PDF ou gráficos sobre energia gerada, consumo e economia financeira são salvos em buckets do S3. Os usuários podem acessar esses relatórios por meio de links gerados pela aplicação, garantindo facilidade de download.
  • Distribuição de Conteúdo Multimídia: Arquivos como vídeos educativos e imagens utilizados na interface do sistema são armazenados no S3, garantindo acesso rápido e eficiente. A integração com o CloudFront melhora o desempenho na entrega de conteúdos em diferentes regiões.

Comunicação em Tempo Real

Tecnologia de Websockets

Para interações em tempo real, como chats ou atualizações dinâmicas, o sistema utiliza Websockets, permitindo uma comunicação bidirecional eficiente entre clientes e servidor. Essa abordagem melhora a responsividade e enriquece a experiência do usuário.

Acessibilidade

Recursos de Inclusão

O sistema foi projetado com recursos que garantem acessibilidade, como:

  • Uso de tags semânticas HTML para melhorar a compatibilidade com leitores de tela.
  • Garantia de contrastes adequados entre texto e fundo para facilitar a leitura.
  • Adaptação do texto com tamanhos ajustáveis usando unidades relativas.
  • Inclusão de textos alternativos para imagens e legendas em conteúdos multimídia.
  • Testes regulares com leitores de tela para garantir total acessibilidade.
